Resume Codex sessions instantly.

cxresume is a focused CLI/TUI that discovers conversation logs under ~/.codex/sessions, previews their content, and launches codex resume <sessionId> so you can return to deep work without hunting for IDs.

Install via npm View on GitHub

Built for fast recovery

Workspace awareness

cxresume cwd keeps a lightweight .cxresume_sessions log in each project so you can focus on the conversations that matter to the current repo.

Split-pane TUI

Browse your Codex history with an interactive picker that previews recent dialogue and launches the session with a single keystroke.

Search-friendly

Filter by directory, content, or recency, or jump straight to a known session file with the --open flag.

Zero setup

No project scaffolding required—point cxresume at your Codex logs and it does the rest. Optional config gives you control when you need it.

Quick start

npm install -g cxresume

# browse entire Codex history
cxresume

# focus on the current workspace sessions
cxresume cwd

# create and launch a fresh workspace session
cxresume cwd -n

How it works

cxresume reads session_meta and event_msg entries from your local Codex session logs to reconstruct message timelines. It extracts the sessionId, shows a live preview, and then invokes codex resume <sessionId> (plus any extra flags you provide). Everything stays on your machine—no cloud sync, no third-party APIs.

Keyboard-first experience

  • Navigate: arrow keys, j/k, ←/→ for paging
  • Launch: press Enter to resume the highlighted session
  • Workspace flow: use s (in cwd mode) to create and resume a new session immediately
  • Cleanup: press d to remove stale logs once you no longer need them
